Overview

What is NashCab?

We model a ride-sharing platform (Uber/Lyft) as a Mean-Field Game (MFG). Each driver is a strategic agent who, at every minute, decides whether to accept or reject a randomly offered ride. Drivers don't reposition proactively — their zone changes only when they complete a trip.

We find the Mean-Field Nash Equilibrium: a joint policy from which no driver can unilaterally deviate to earn more, given all other drivers follow the same strategy.

🎯

Backward Induction

Compute V(z,t) — expected future earnings — backwards from T→0. At each step, the driver compares Q_accept vs Q_reject using a sigmoid policy.

🔄

Forward Simulation

Run all drivers forward using the computed policy π. Drivers scatter to destination zones after accepting rides, producing a new µ_idle distribution.

⚖️

Nash Equilibrium

When µ_new ≈ µ_old, we have found a fixed point. Every driver is already best-responding — no one can do better by deviating.

📉

Price of Anarchy

PoA = W_opt / W_MFNE measures welfare loss from selfish behavior. A PoA of 1.52 means drivers deliver only 66% of the socially optimal matching.

We start with every driver spread uniformly — 1/500 of drivers at each of the 49 zones at t=0. We can't just hold this uniform distribution for all T steps because by t=250, drivers have been moving for 250 steps and the real distribution looks nothing like uniform. Using uniform throughout would make the first backward pass compute p_offer values ~3–4× off at mid-horizon, and the outer loop never recovers. So we first run one forward simulation under an always-accept policy to get a physically consistent starting trajectory, then begin the fictitious play loop from there.

Convergence

All three profiles converge with α = 0.05 and ε = 0.01. Log-linear L1 decay consistent with geometric convergence.

Two differences from Phase 1. First, the starting distribution at t=0 is not uniform — drivers are placed proportional to total daily pickups per zone, so high-traffic zones like Midtown start with more drivers than quiet outer-borough zones. Second, Phase 2 also needs to initialize the 5-minute ride pool R[z,t] (the age-cohort backlog of unaccepted rides), not just µ. The always-accept warm start produces both together, giving the outer loop a physically consistent starting point for both the driver distribution and the ride pool.

Convergence

Converged at iteration 166 with final L1 = 0.98. α=0.02 (smaller than Phase 1 to handle spiky real demand). ε=1.0 calibrated to the integer noise floor of real trip counts.

Why is Real-World PoA Higher?

Real NYC demand is spiky — airports and Midtown generate a disproportionate share of trips. This creates stronger clustering incentives, widening the gap between selfish and optimal matching.

🔧

Why Smaller α and Larger ε?

Integer trip counts create a noise floor in the forward simulation. A larger α amplifies this noise causing oscillation. ε=1.0 is calibrated to this floor — it represents genuine convergence, not a loose threshold.

🏙️

34% Welfare Loss

PoA = 1.52 means the selfish equilibrium delivers only 66% of the socially optimal ride matching. A central planner could match 52% more rides with the same number of drivers.

What is a Mean-Field Game?

A way to find Nash equilibria when there are too many players to track individually.

The Core Approximation
N players → O(N²) interactions to track
↓  MFG approximation
You only need to track µ, not every other driver
Finding the Equilibrium
µ⁽ᵏ⁾  (mean field)
↓ backward induction → V, π
π*  (optimal policy)
↓ forward simulate all drivers
µ̂⁽ᵏ⁺¹⁾  (new distribution)
↓ smooth mix: α·µ̂ + (1−α)·µ
converged?  ‖µ_new − µ‖₁ < ε
✓  At convergence: µ* is self-consistent.
Playing optimally against µ* reproduces µ*.
